# 🏔 Project 100 Principles

These principles guide every design decision.

If a feature conflicts with these principles, the feature changes.

The principles do not.

---

# 1. Calm before clever

Simple solutions are preferred over impressive ones.

Project 100 should feel peaceful.

---

# 2. One card. One purpose.

Each card answers one question.

If it answers two questions, split it.

---

# 3. Mindset before metrics

Purpose comes before performance.

The athlete should understand why today's expedition matters before seeing any numbers.

---

# 4. Software should disappear

The goal is not to spend time inside Project 100.

The goal is to spend time outside.

---

# 5. Build confidence, not dependence.

Project 100 should increase the athlete's confidence.

It should never encourage unnecessary engagement.

---

# 6. Every feature earns its place.

No feature exists simply because another application has it.

Every feature must improve today's expedition.

---

# 7. Today's expedition matters most.

Tomorrow's races are built from today's decisions.

Project 100 therefore focuses on today's expedition above everything else.

---

# 8. Every summit reveals another horizon.

Completion is never the end.

It is another milestone.

Project Beyond exists as a reminder that adventure continues.

---

# 9. Design for the mountains.

Whenever a design decision is difficult, ask:

"Would this feel at home in a National Park visitor centre?"

If the answer is no, rethink it.

---

# 10. One purposeful step today.

This is the heart of Project 100.

The application exists to encourage one more purposeful step.
