# 🎨 Project 100 Design System

Version 1.0

---

# Purpose

The Project 100 design system exists to ensure every screen feels like part of the same expedition.

Users should never notice the design.

They should simply feel calm.

---

# Design Inspiration

Project 100 takes inspiration from:

- National Park visitor centres
- Ordnance Survey maps
- Expedition journals
- Scandinavian design
- Apple Human Interface Guidelines

# Typography

Three font sizes only.

## Heading

Large

Used for greetings and expedition titles.

---

## Section

Medium

Used for card headings.

---

## Body

Standard

Everything else.

---

# Card Principles

Every card answers one question.

Never combine unrelated information.

Cards should breathe.

Whitespace is part of the design.

---

# Animation

Animations should feel natural.

Maximum duration:

250 ms

No bouncing.

No exaggerated effects.

---

# Icons

Icons should be simple line drawings.

Preferred subjects:

- Mountain
- Trail
- Runner
- Walking boot
- Dumbbell
- Apple
- Water bottle
- Camp
- Cairn
- Compass

Avoid colourful icons.

---

# Buttons

Buttons should encourage action without demanding attention.

Primary actions use Trail Blue.

Success states use Trail Green.

---

# Philosophy

The interface should disappear quickly.

The athlete should remember the expedition rather than the software.
