It exists to keep the product coherent as it grows.

# Release Strategy

Every release must include:

- Updated documentation
- Updated changelog
- Expedition log entry
- Testing checklist

---

# Definition of Done

A feature is complete when:

- Code is complete.
- Documentation updated.
- Tested.
- Release notes written.
- Approved.

# Philosophy

Quality before speed.

Thoughtfulness before quantity.
